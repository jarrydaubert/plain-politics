# Source Reference Policy

## Principle

No source means no factual claim. The product should prefer a clear "cannot verify" or coverage-gap state over a polished but unsupported statement.

## Source Reference Requirements

Every factual summary, chart, table, map, and comparison should include:

- At least one valid source reference for each material factual section.
- Claim-to-source mapping for each material factual claim.
- Source title, publisher, URL, retrieved date, and source tier.
- Evidence snippet, source excerpt, or source span sufficient for review.
- Page, section, or table reference when the source format supports it.

## Unsupported Evidence

If available sources cannot support a claim, the page should:

1. Say that the claim cannot be verified from available sources.
2. Avoid filling gaps with editorial guesswork.
3. Link the closest verified source or explain the coverage gap.
4. Avoid naming a party, person, donor, or organisation as responsible unless a source directly supports that statement.

## Source Quality Checks

Before release, test that:

1. Source URLs resolve or have stored snapshots.
2. Cited snippets or excerpts support the claim they are attached to.
3. No factual sentence is orphaned from evidence.
4. Higher-tier sources are preferred when multiple sources support the same claim.
5. Stale sources are labelled according to freshness policy.

## User Interface Requirements

- Show source references close to the claims they support.
- Show source tier and last verified date where space allows.
- Distinguish plain-English summaries from quoted source text.
- Make source links keyboard accessible and screen-reader understandable.
